Introduction to SQL

SQL (Structured Query Language) is a standard programming language designed for managing and manipulating relational databases. It allows you to create, read, update, and delete data from databases. SQL is declarative, meaning you specify what you want, not how to get it. It's widely used across different database management systems like MySQL, PostgreSQL, SQL Server, Oracle, and SQLite.

Basic SQL Syntax Rules:

-- Comments start with double dashes
/* Multi-line comments 
   use slash-asterisk */

-- SQL statements end with semicolon
SELECT * FROM users;

-- SQL is case-insensitive for keywords
select * from USERS;  -- Same as above

-- Good practice: Use uppercase for keywords
SELECT name FROM users WHERE age > 18;

Basic SQL Commands

SQL commands are categorized into different types: DDL (Data Definition Language), DML (Data Manipulation Language), DCL (Data Control Language), and TCL (Transaction Control Language). Understanding these categories helps you grasp the purpose and scope of each command.

DDL (Data Definition Language):

-- CREATE: Create database objects
CREATE DATABASE my_company;
CREATE TABLE employees (id INT, name VARCHAR(100));

-- ALTER: Modify existing database objects
ALTER TABLE employees ADD COLUMN email VARCHAR(255);
ALTER TABLE employees MODIFY COLUMN name VARCHAR(150);

-- DROP: Delete database objects
DROP TABLE employees;
DROP DATABASE my_company;

-- TRUNCATE: Remove all data from table (structure remains)
TRUNCATE TABLE employees;

DML (Data Manipulation Language):

-- INSERT: Add new data
INSERT INTO employees (name, email) VALUES ('John Doe', 'john@email.com');

-- SELECT: Retrieve data
SELECT * FROM employees;

-- UPDATE: Modify existing data
UPDATE employees SET email = 'john.doe@email.com' WHERE name = 'John Doe';

-- DELETE: Remove data
DELETE FROM employees WHERE name = 'John Doe';

Data Types

SQL data types define the kind of data that can be stored in table columns. Choosing appropriate data types is crucial for database performance, storage efficiency, and data integrity.

Numeric Data Types:

-- Integer types
CREATE TABLE numeric_examples (
    tiny_num TINYINT,        -- -128 to 127
    small_num SMALLINT,      -- -32,768 to 32,767
    medium_num MEDIUMINT,    -- -8,388,608 to 8,388,607
    regular_num INT,         -- -2,147,483,648 to 2,147,483,647
    big_num BIGINT          -- Very large range
);

-- Decimal types
CREATE TABLE decimal_examples (
    price DECIMAL(10,2),     -- 10 digits total, 2 after decimal
    percentage FLOAT,        -- Floating point
    precise_calc DOUBLE      -- Double precision floating point
);

String Data Types:

-- Character types
CREATE TABLE string_examples (
    short_text CHAR(10),     -- Fixed length
    variable_text VARCHAR(255), -- Variable length
    long_text TEXT,          -- Large text
    binary_data BLOB         -- Binary data
);
